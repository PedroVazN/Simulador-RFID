Integração RFID × RH × MES (simulado)

Este painel imita o que acontece na fábrica: o leitor de crachá envia a tag RFID, o servidor (como o MES) pergunta ao RH do ERP se aquele colaborador tem treinamento com status Ativo para aquela máquina. Só então o MES sinalizaria Start_Permit para o CLP começar o ciclo. Aqui, tudo é fictício, mas a ordem das decisões é a mesma.

Como o fluxo é avaliado (em ordem)

  1. É lida a tag do crachá (você digita o código para simular a leitura).
  2. É verificado se a tag existe no cadastro de RH (simulado neste demo).
  3. O status do treinamento precisa ser exatamente Ativo (outros status bloqueiam).
  4. A máquina selecionada precisa estar na lista de máquinas que o colaborador pode operar com aquele treinamento.
  5. Se tudo passar, o retorno indica Start_Permit = sim (no projeto real, isso iria ao CLP via MES/OPC).
